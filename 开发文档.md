# 家具软装内部管理平台 — 开发文档

## 1. 技术栈

| 层级 | 技术 | 版本 |
|------|------|------|
| 后端框架 | Django + Django REST Framework | Django 5.x, DRF 3.15+ |
| 数据库 | PostgreSQL | 16 |
| 前端框架 | React + TypeScript | React 18, TS 5 |
| UI 组件库 | Ant Design（包豪斯风格定制主题） | 5.x |
| 前端构建 | Vite | 5.x |
| 状态管理 | Zustand | 4.x |
| HTTP 客户端 | Axios | 1.x |
| 路由 | React Router | 6.x |
| Web 服务器 | Nginx（前端）+ Gunicorn（后端） | - |
| 容器化 | Docker Compose | - |
| 测试部署 | Vercel（前端）+ Railway（后端+数据库） | - |
| CI/CD | GitHub → Vercel/Railway 自动部署 | - |
| PDF 生成 | WeasyPrint | 62.x |
| Excel 解析 | openpyxl | 3.x |
| 图片处理 | Pillow | 10.x |

## 3. 数据模型

### 3.1 用户模型 (auth_app.User)

```
User (继承 AbstractUser)
├── username        VARCHAR(150)    用户名（唯一）
├── password        VARCHAR(128)    密码（哈希）
├── role            VARCHAR(10)     角色：ADMIN/STAFF
├── display_name    VARCHAR(100)    显示名称
├── is_active       BOOLEAN         是否启用
└── date_joined     DATETIME        注册时间
```

### 3.2 产品模型

```
Category（产品分类，树形结构）
├── id              BIGINT PK
├── name            VARCHAR(100)    分类名称
├── parent          FK → Category   父分类（NULL=顶级）
├── dimension       VARCHAR(10)     维度：TYPE/SPACE/ORIGIN
├── sort_order      INT             排序序号
└── unique_together: (parent, name, dimension)

Product（产品）
├── id              BIGINT PK
├── name            VARCHAR(200)    产品名称
├── code            VARCHAR(50)     产品编号（唯一，可空）
├── description     TEXT            产品描述
├── category        FK → Category   主分类
├── origin          VARCHAR(10)     产地：IMPORT/DOMESTIC/CUSTOM
├── min_price       DECIMAL(10,2)   最低售价
├── is_active       BOOLEAN         是否上架
├── created_by      FK → User       创建人
├── categories      M2M → Category  关联分类（通过 ProductCategory）
├── created_at      DATETIME
└── updated_at      DATETIME

ProductImage（产品图片）
├── id              BIGINT PK
├── product         FK → Product
├── image_path      VARCHAR(500)    图片相对路径
├── thumbnail_path  VARCHAR(500)    缩略图路径
├── sort_order      INT             排序
├── is_cover        BOOLEAN         是否封面
└── created_at      DATETIME

ProductConfig（产品配置）
├── id              BIGINT PK
├── product         FK → Product
├── config_name     VARCHAR(200)    配置名称
├── attributes      JSON            自定义属性 {"颜色":"黑色","尺寸":"L"}
├── guide_price     DECIMAL(10,2)   指导价格
└── created_at      DATETIME
```

### 3.3 案例模型

```
Case（客户案例）
├── id              BIGINT PK
├── title           VARCHAR(200)    案例标题
├── description     TEXT            项目描述
├── industry        VARCHAR(20)     行业分类（9种）
├── created_by      FK → User
├── products        M2M → Product   关联产品（通过 CaseProduct）
├── created_at      DATETIME
└── updated_at      DATETIME

CaseImage（案例图片）
├── id, case(FK), image_path, thumbnail_path, sort_order, is_cover, created_at
```

### 3.4 文档模型

```
DocumentFolder（文件夹，树形结构）
├── id              BIGINT PK
├── name            VARCHAR(200)
├── doc_type        VARCHAR(15)     DESIGN/TRAINING/CERTIFICATE
├── parent          FK → self
├── sort_order      INT
└── created_at      DATETIME

Document（文档文件）
├── id              BIGINT PK
├── name            VARCHAR(300)    文件名
├── doc_type        VARCHAR(15)     文档类型
├── folder          FK → DocumentFolder
├── file_path       VARCHAR(500)    文件相对路径
├── file_size       BIGINT          文件大小（字节）
├── mime_type       VARCHAR(100)    MIME 类型
├── tags            JSON            标签数组 ["标签1","标签2"]
├── created_by      FK → User
└── created_at      DATETIME
```

### 3.5 报价模型

```
Quote（报价单）
├── id              BIGINT PK
├── title           VARCHAR(200)    报价单标题
├── customer_name   VARCHAR(200)    客户名称
├── status          VARCHAR(15)     DRAFT/SENT/CONFIRMED/CANCELLED
├── notes           TEXT            备注
├── terms           TEXT            条款
├── total_amount    DECIMAL(12,2)   总金额（自动计算）
├── created_by      FK → User
├── created_at      DATETIME
└── updated_at      DATETIME

QuoteItem（报价明细）
├── id              BIGINT PK
├── quote           FK → Quote
├── product         FK → Product    关联产品（可空）
├── product_name    VARCHAR(200)    产品名称（快照）
├── config_name     VARCHAR(200)    配置名称
├── unit_price      DECIMAL(10,2)   单价
├── quantity        INT             数量（≥1）
├── discount        DECIMAL(5,2)    折扣百分比（0-100）
├── subtotal        DECIMAL(12,2)   小计 = 单价 × 数量 × (1 - 折扣/100)
└── sort_order      INT
```

### 3.6 分享模型

```
ShareLink（分享链接）
├── id              BIGINT PK
├── token           VARCHAR(64)     UUID，唯一索引
├── content_type    VARCHAR(10)     PRODUCT/CASE/QUOTE/CATALOG
├── object_id       INT             关联对象 ID
├── title           VARCHAR(200)    分享标题
├── password_hash   VARCHAR(128)    密码哈希（空=无密码）
├── expires_at      DATETIME        过期时间（NULL=永不过期）
├── max_access_count INT            最大访问次数（NULL=无限制）
├── access_count    INT             已访问次数
├── is_active       BOOLEAN         是否启用
├── created_by      FK → User
└── created_at      DATETIME

ShareAccessLog（访问记录）
├── id, share_link(FK), accessed_at, ip_address, user_agent
```

## 6. 认证机制

- 使用 Django REST Framework 的 Token Authentication
- 登录成功后返回 token，前端存储在 localStorage
- 每次请求在 Header 中携带 `Authorization: Token {token}`
- 401 响应自动清除 token 并跳转登录页

## 7. 文件存储

- 上传文件存储在 `MEDIA_ROOT` 目录下
- 图片上传后自动生成缩略图（small: 150×150, medium: 400×400）
- 文件路径格式：`{subdirectory}/{YYYYMMDD}/{uuid}_{filename}`
- 仅支持本地磁盘存储（不使用云对象存储）
- 测试环境（Railway）：使用 Railway 持久化存储卷
- 生产环境（NAS）：Docker volume 挂载到 NAS 本地磁盘

## 8. 部署架构

### 8.1 测试环境：Vercel + Railway（GitHub 自动部署）

```
GitHub Repository
  ├── 推送代码 → 自动触发部署
  │
  ├── Vercel（前端）
  │   ├── 自动构建 React 项目（npm run build）
  │   ├── 静态文件托管 + CDN 加速
  │   ├── 环境变量: VITE_API_BASE_URL=https://<railway-backend>.railway.app
  │   └── 自定义域名（可选）
  │
  └── Railway（后端 + 数据库）
      ├── Django + Gunicorn
      │   ├── 自动检测 Python 项目并部署
      │   ├── 启动命令: gunicorn config.wsgi --bind 0.0.0.0:$PORT
      │   ├── 静态文件: WhiteNoise 中间件直接服务
      │   ├── 媒体文件: Railway 持久化存储卷
      │   └── 环境变量: DATABASE_URL, DJANGO_SECRET_KEY, CORS 等
      └── PostgreSQL 插件
          ├── Railway 内置 PostgreSQL 服务
          └── 自动注入 DATABASE_URL 环境变量
```

**测试环境特点**：
- 推送到 GitHub main 分支自动部署，无需手动操作
- 前后端分离部署，前端通过环境变量配置 API 地址
- Railway 提供免费的 PostgreSQL 数据库
- 适合开发测试和演示

### 8.2 生产环境：Synology NAS Docker Compose

```
docker-compose.yml 定义 3 个服务：

db (postgres:16-alpine)
  ├── 端口: 5432
  ├── 数据: postgres_data volume
  └── 环境变量: POSTGRES_DB, POSTGRES_USER, POSTGRES_PASSWORD

  ├── 端口: 8000
  ├── 启动: entrypoint.sh → makemigrations → migrate → create_admin → gunicorn
  ├── 数据: media_data, static_data volumes
  └── 依赖: db

frontend (React + Nginx)
  ├── 端口: 80
  ├── Nginx 配置:
  │   ├── / → 前端静态文件
  │   ├── /api/* → 反向代理到 backend:8000
  │   ├── /media/* → 媒体文件目录
  │   └── /static/* → Django 静态文件
  └── 依赖: backend
```

**生产环境特点**：
- 部署在客户 Synology DS920+ NAS 上
- 局域网直接访问 + 外网通过 QuickConnect/端口映射/DDNS 访问
- Docker Compose 一键部署，容器设置 restart: always 开机自启
- 媒体文件通过 NAS Hyper Backup 备份

## 9. 关键业务逻辑

### 9.1 报价单金额计算
- 每个明细项：`subtotal = unit_price × quantity × (1 - discount / 100)`
- 总金额：所有明细项 subtotal 之和
- 在 QuoteItem.save() 中自动计算 subtotal
- 在增删改明细后调用 Quote.recalculate_total() 更新总金额

### 9.2 分享链接验证流程
1. 前端访问 `/s/{token}`
2. 调用 `GET /api/share/{token}/` 获取分享信息
3. 如果需要密码，显示密码输入框
4. 调用 `POST /api/share/{token}/verify/` 验证密码
5. 验证通过后显示分享内容，同时记录访问日志

### 9.3 产品搜索逻辑
搜索关键词同时匹配以下字段（OR 关系）：
- 产品名称 (name)
- 产品编号 (code)
- 产品描述 (description)
- 配置名称 (configs__config_name)
- 配置属性 (configs__attributes)

### 9.4 分类树查询
- 按维度（TYPE/SPACE/ORIGIN）查询顶级分类
- 递归序列化子分类，返回完整树结构
- 产品筛选时，选中某分类会包含其所有子分类下的产品

## 10. 环境变量

### 10.1 通用环境变量

| 变量 | 说明 | 默认值 |
|------|------|--------|
| DJANGO_SECRET_KEY | Django 密钥 | dev-secret-key |
| DJANGO_DEBUG | 调试模式 | True |
| DJANGO_ALLOWED_HOSTS | 允许的主机名 | localhost,127.0.0.1 |
| CORS_ALLOWED_ORIGINS | CORS 允许的源 | http://localhost |
| ADMIN_USERNAME | 初始管理员用户名 | admin |
| ADMIN_PASSWORD | 初始管理员密码 | admin123456 |

### 10.2 数据库环境变量

| 变量 | 说明 | 默认值 |
|------|------|--------|
| DATABASE_URL | 数据库连接URL（Railway 优先使用） | - |
| DB_NAME | 数据库名（Docker 部署使用） | furniture_app |
| DB_USER | 数据库用户 | postgres |
| DB_PASSWORD | 数据库密码 | postgres |
| DB_HOST | 数据库主机 | localhost |
| DB_PORT | 数据库端口 | 5432 |

### 10.3 存储环境变量

| 变量 | 说明 | 默认值 |
|------|------|--------|
| MEDIA_ROOT | 媒体文件目录 | /app/media |

### 10.4 前端环境变量（Vercel）

| 变量 | 说明 | 默认值 |
|------|------|--------|
| VITE_API_BASE_URL | 后端 API 地址 | http://localhost:8000 |

**说明**：
- Railway 部署时优先使用 `DATABASE_URL`（Railway 自动注入），如不存在则使用 `DB_*` 系列变量

- 媒体文件统一使用本地磁盘存储（测试环境使用 Railway 持久化卷，生产环境使用 NAS 本地磁盘）
