# 家具软装内部管理平台 — 需求文档

## 1. 项目概述

### 1.1 项目背景
家具软装公司内部使用的资料管理系统，用于管理产品信息、客户案例、内部文档、报价方案，并支持通过分享链接将产品/报价信息发送给外部客户查看。

### 1.2 目标用户
- **管理员**：公司管理层，拥有全部权限（增删改查、用户管理）
- **普通员工**：销售/设计人员，拥有查看权限和部分创建权限
- **外部客户**：通过分享链接访问，无需注册登录

### 1.3 部署环境

#### 测试环境
- **前端**: Vercel（自动从 GitHub 部署，CDN 加速）
- **后端**: Railway（Django + Gunicorn + PostgreSQL）
- **CI/CD**: 推送到 GitHub main 分支后自动部署到 Vercel 和 Railway
- **用途**: 开发测试和功能验证

#### 生产环境
- **目标硬件**: Synology DS920+ NAS
- **部署方式**: Docker Compose 一键部署
- **访问方式**: 局域网直接访问 + 外网通过 QuickConnect/端口映射/DDNS 访问

## 2. 功能模块

### 2.1 用户认证与权限

| 功能 | 说明 |
|------|------|
| 登录/登出 | 用户名+密码登录，Token 认证 |
| 角色体系 | 管理员（ADMIN）、普通员工（STAFF）两种角色 |
| 权限控制 | 管理员可增删改查所有数据；员工只能查看，不能删除 |
| 用户管理 | 管理员可创建/编辑/启用/禁用用户账号 |
| 初始账号 | 系统启动时自动创建管理员账号（来自环境变量） |

### 2.2 产品管理

| 功能 | 说明 |
|------|------|
| 产品 CRUD | 创建/编辑/删除/查看产品，包含名称、编号、产地、描述、最低售价 |
| 产地分类 | 进口、国产、定制三种产地类型 |
| 三维度分类 | 按类型、按空间、按产地三个维度的树形分类体系 |
| 分类拖拽排序 | 支持拖拽调整分类的排列顺序 |
| 多分类关联 | 一个产品可关联多个分类 |
| 产品图片 | 支持多图上传、封面设置、排序、缩略图自动生成 |
| 多配置价格 | 每个产品可有多个配置（如不同尺寸/材质），每个配置有独立的指导价格和自定义属性（JSON） |
| 全字段搜索 | 搜索范围覆盖产品名称、编号、描述、配置名称、配置属性 |
| 上架/下架 | 管理员可控制产品是否对员工可见 |
| 批量导入 | 支持通过 Excel 文件批量导入产品（模板下载+上传解析） |
| 相关资源链接 | 产品详情页可快速跳转到设计资源、培训资料、资质文件 |

### 2.3 产品图册

| 功能 | 说明 |
|------|------|
| 图册浏览 | 以卡片形式展示所有上架产品，含封面图、名称、编号 |
| 分类导航 | 左侧树形分类导航，点击筛选对应分类下的产品 |
| 搜索 | 按名称/编号搜索产品 |
| 分页 | 分页加载，避免大量数据卡顿 |

### 2.4 客户案例

| 功能 | 说明 |
|------|------|
| 案例 CRUD | 创建/编辑/删除/查看客户案例 |
| 行业分类 | 9 大行业分类：科技/互联网、金融/保险/财税、地产/建筑/设计院、教育培训、医疗/大健康、广告/文创/传媒、制造/实业/工厂、政府/国企/事业单位、其他 |
| 案例图片 | 支持多图上传、封面设置、缩略图 |
| 关联产品 | 案例可关联多个产品，详情页展示关联产品信息 |
| 行业筛选 | 列表页按行业筛选案例 |

### 2.5 内部文档

| 功能 | 说明 |
|------|------|
| 三种文档类型 | 设计资源（DESIGN）、培训资料（TRAINING）、资质文件（CERTIFICATE） |
| 文件夹管理 | 树形文件夹结构，支持创建/删除文件夹 |
| 文件上传 | 支持上传任意类型文件，最大 50MB |
| 文件下载 | 点击下载按钮直接下载文件 |
| 在线预览 | 支持图片、PDF、音频、视频的在线预览/播放 |
| 文件搜索 | 按文件名搜索 |
| 标签系统 | 文件可添加标签，按标签筛选 |

### 2.6 报价方案

| 功能 | 说明 |
|------|------|
| 报价单 CRUD | 创建/编辑/删除报价单，包含标题、客户名称、备注、条款 |
| 报价明细 | 从产品库搜索添加产品到报价单，支持选择配置 |
| 价格计算 | 每项可设置单价、数量、折扣(%)，自动计算小计和总金额 |
| 状态管理 | 草稿 → 已发送 → 已确认 → 已取消 |
| 复制报价单 | 一键复制现有报价单（含所有明细） |
| PDF 导出 | 生成 PDF 格式的报价单文件 |
| 搜索筛选 | 按标题/客户名搜索，按状态筛选 |

### 2.7 分享功能

| 功能 | 说明 |
|------|------|
| 创建分享链接 | 可分享产品、案例、报价单、产品图册 |
| 访问控制 | 可设置密码保护、过期时间、最大访问次数 |
| 启用/禁用 | 可随时启用或禁用分享链接 |
| 公开访问 | 外部客户通过链接直接访问，无需登录 |
| 访问记录 | 记录每次访问的时间、IP、浏览器信息 |
| 分享链接格式 | `http://域名/s/{uuid}` |

### 2.8 全局搜索

| 功能 | 说明 |
|------|------|
| 跨模块搜索 | 同时搜索产品、案例、文档、报价单 |
| 实时搜索 | 顶部搜索栏输入关键词，下拉展示结果 |
| 结果跳转 | 点击搜索结果直接跳转到对应详情页 |

## 3. 非功能需求

| 项目 | 要求 |
|------|------|
| 响应时间 | 页面加载 < 2 秒，API 响应 < 500ms |
| 文件大小 | 图片上传最大 10MB，文档上传最大 50MB |
| 缩略图 | 图片上传后自动生成 small(150×150)、medium(400×400) 缩略图 |
| 数据安全 | 密码加密存储，Token 认证，分享链接密码哈希存储 |
| 数据库 | 仅支持 PostgreSQL 16（测试环境 Railway 提供，生产环境 Docker 容器） |
| 部署方式 | 测试环境：Vercel（前端）+ Railway（后端+数据库），GitHub 自动部署；生产环境：Synology NAS Docker Compose 一键部署 |
| CI/CD | 推送到 GitHub main 分支后自动部署到测试环境（Vercel + Railway 原生支持） |
| 开机自启 | 生产环境 Docker 容器设置 restart: always |
| 数据备份 | 生产环境：支持数据库导出/恢复，媒体文件可通过 NAS Hyper Backup 备份 |
| 媒体文件存储 | 仅使用本地磁盘存储（测试环境 Railway 持久化卷，生产环境 NAS 本地磁盘） |
| 静态文件服务 | 测试环境：Vercel 托管前端静态文件，Railway 通过 WhiteNoise 服务 Django 静态文件；生产环境：Nginx 统一代理 |
| UI 设计风格 | 包豪斯（Bauhaus）风格定制主题：几何形态、功能主义、极简美学；微交互动效（按钮缩放、卡片悬浮、页面过渡）；微阴影系统（Soft Shadow，超长弥散半径柔和阴影）；自定义字体（DM Sans + Inter + Noto Sans SC） |
